# PRD: idea2ui — v4.0

## 1. 产品概述

**idea2ui** 帮助用户将一个想法迅速变为一个可用的 UI 交付。用户通过自然语言描述需求，AI 自动生成原生 Web 原型（HTML + CSS + JS），并支持实时预览、拖拽调整和迭代优化。

## 2. 核心流程

### 2.1 项目创建

1. **新建项目** — 用户点击「新建」，选择目标平台（Web / 移动网页 / App / 小程序）
2. **输入项目名称** — 后端创建项目文件夹，所有数据持久化到磁盘

### 2.2 PRD 先行

1. **PRD 设计** — AI 引导用户对话完成 PRD 设计，PRD 中需包含完整的页面结构描述
2. **PRD 预览确认** — 生成的 PRD 文档在预览界面展示（Markdown 渲染），用户确认后进入下一阶段
3. **页面导航生成** — 根据确认后的 PRD 页面结构，在预览下方生成页面导航标签
4. **页面生成** — 用户点击导航中某个页面：
   - 若页面尚未生成，提示用户输入该页面的具体需求，AI 根据 PRD + 用户输入生成该页面
   - 若页面已生成，直接渲染预览
5. **实时预览 + 设备模拟** — 预览面板提供设备尺寸模拟（PC / iPad / Phone / 自定义），用户可切换不同尺寸预览
6. **结果导出** — 将调整后的结果写入产出文件夹
7. **持续迭代** — 后续对话携带已调整版本作为上下文

### 2.3 项目切换

- 用户可通过顶部项目选择器切换项目
- 每个项目独立保存对话记录、PRD、页面代码
- 重启服务后数据不丢失

## 5. 技术方向

### 产出
- 页面输出格式：始终为原生 HTML + CSS + JS（不依赖任何框架）
- PRD 格式：Markdown，包含页面结构列表
- 存储方式：每个项目独立文件夹，JSON 元数据 + 代码文件分离

### AI 生成
- AI 集成：通过 LLM API 生成代码
- 双模式：PRD 设计模式（输出 Markdown）和页面生成模式（输出 HTML/CSS/JS）
- 预览引擎：iframe / 沙箱渲染
- 拖拽方案：原生拖拽 API 或轻量库（如 interact.js）

### 设备模拟
- 预设尺寸：根据平台类型提供不同预设列表
  - Web：1440×900, 1920×1080, 1024×768
  - 移动网页：375×812, 390×844, 414×896
  - App：375×812, 390×844, 430×932
  - 小程序：375×812, 390×844
- 自定义：支持手动输入宽高
- 展示：iframe 外嵌设备边框（圆角+阴影），小屏设备显示状态栏

### 项目本身技术栈
- 形态：纯 Web 应用
- 前端框架：Vue
- 构建工具：Vite
- 语言：TypeScript
- UI 组件库：antd
- 项目自身样式方案：CSS Modules 或 Tailwind CSS

## 6. 后端架构

### 技术栈
- 语言：Python
- Web 框架：FastAPI
- HTTP 客户端：httpx（用于调用 LLM API）
- 运行方式：Uvicorn ASGI 服务器

### 接口设计

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/chat | 发送对话消息，关联 project_id，返回 AI 生成结果 |
| POST | /api/export | 导出生成的 UI 文件到输出目录 |
| GET | /api/projects | 列出所有持久化项目 |
| POST | /api/projects | 创建新项目（含名称和平台类型） |
| GET | /api/projects/{id} | 读取指定项目的完整数据 |
| PUT | /api/projects/{id} | 更新项目数据（消息/页面/PRD） |
| DELETE | /api/projects/{id} | 删除项目 |
| GET | /api/config | 读取模型配置 |
| POST | /api/config | 保存模型配置 |
| GET | /api/health | 健康检查 |

### AI 集成流程

1. 前端收集用户消息 + 对话历史 + 模型配置（API Key、模型名、Base URL）
2. 前端通过 `/api/chat` 发送给后端
3. 后端根据当前阶段（PRD 设计 or 页面生成）构造对应 system prompt，调用 LLM API
4. LLM 返回结构化数据
5. 后端解析后返回给前端
6. 前端更新对话列表和预览面板

### 上下文管理
- 每次对话携带完整消息历史
- 若用户已有手动调整的版本，作为 `modified_code` 字段发送给 LLM

### 数据持久化
- 数据路径：`output/{project_id}/project.json`
- 存储内容：项目元数据、完整消息历史、PRD 内容、页面代码
- 页面代码同时单独输出为 `output/{project_id}/pages/{page_name}/` 目录下的独立文件
- 页面切换时自动从磁盘加载项目数据

### 文件导出
- 导出路径通过 `IDEA2UI_OUTPUT_DIR` 环境变量配置（默认 `output/`）
- 每个项目导出为独立文件夹

### 目录结构

```
server/
├── main.py               # 应用入口，CORS 配置
├── config.py             # 环境变量配置
├── requirements.txt      # Python 依赖
├── models/
│   └── schemas.py        # Pydantic 请求/响应模型
├── routers/
│   ├── chat.py           # AI 对话接口（关联 project_id）
│   ├── files.py          # 文件导出接口
│   ├── config.py         # 配置读写接口
│   └── projects.py       # 项目 CRUD 接口
└── services/
    ├── llm.py            # LLM API 调用（PRD/Page 双模式）
    ├── file_manager.py   # 文件导出管理
    ├── config_manager.py # 模型配置持久化
    └── project_manager.py# 项目数据持久化（JSON 文件存储）
```
